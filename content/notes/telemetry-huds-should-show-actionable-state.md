---
title: "Telemetry HUDs should show actionable state"
date: 2026-03-21
tags: ["telemetry", "racing", "ui"]
description: "A HUD is useful when it exposes the state a driver or analyst can act on."
---

A telemetry HUD should not display data just because it exists.

It should expose state that changes a decision: speed, delta, line, grip margin, braking point, lap segment, or sensor confidence. The interface should reduce interpretation time.

Good telemetry UI turns measurement into action.
